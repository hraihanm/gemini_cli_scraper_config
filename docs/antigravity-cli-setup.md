# Antigravity CLI (`agy`) setup

Google deprecated **Gemini CLI** on **2026-06-18**. This repo runs on **Antigravity CLI** (`agy`).

## Prerequisites

1. Install [Antigravity CLI](https://antigravity.google/docs/gcli-migration) (`agy` on PATH).
2. Log in inside `agy` (Google account).
3. Copy env template and set your API key:

```powershell
Copy-Item .agents\.env.example .agents\.env
# Edit AGY_API_KEY and AGY_MODEL (e.g. gemini-3.5-flash)
```

## One-time machine setup

From the repo root:

```powershell
pwsh -NoProfile -File scripts/setup-agy.ps1
```

Then **restart `agy`** in this directory:

```powershell
cd D:\DataHen\projects\gemini_cli_testbed
agy
```

### What the setup script does

| Step | Why |
|------|-----|
| Copies `.agents/skills/<name>/SKILL.md` → `~/.gemini/antigravity-cli/skills/` | AGY global skills (commands + semantic knowledge) |
| Copies `.agents/skills/<name>/SKILL.md` → `~/.gemini/skills/` | AGY shared global fallback path |
| Copies `.agents/skills/<name>/SKILL.md` → `~/.cursor/skills/` | Cursor global skills (same format, instant `/name` in any project) |
| Stages `.agents/plugins/gemini_cli_testbed/skills/` + `agy plugin install` | Registers playwright-mod MCP + plugin commands |

> Running `agy` from the repo root already auto-discovers workspace `.agents/skills/`. The global sync + plugin install make them available from any directory and register the MCP.

## The two layers (native model)

| Layer | Path | Role | Loaded |
|---|---|---|---|
| **AGENTS.md** | repo root | persona + firmware rules | always (prepended) |
| **Skills** | `.agents/skills/<name>/SKILL.md` | slash commands AND reusable knowledge | on `/<name>` (command skills) or semantic match on `description` (knowledge skills) |

Skills are the only mechanism for both slash commands and knowledge. There is no separate "workflows" layer.

## Layout (do not break)

```
.agents/
  mcp_config.json                  # workspace MCP (playwright-mod)
  .env                             # AGY_API_KEY, AGY_MODEL
  skills/                          # ALL slash commands AND knowledge (one SKILL.md per topic)
    scrape/SKILL.md                # command skill → /scrape
    navigation-parser/SKILL.md     # command skill → /navigation-parser
    datahen-conventions/SKILL.md   # knowledge skill (semantic-loaded)
    ...
  plugins/
    gemini_cli_testbed/            # plugin bundle
      plugin.json                  # manifest (the ONLY plugin.json)
      mcp_config.json
      skills/                      # generated by setup-agy.ps1 (gitignored)

AGENTS.md                          # project context (always prepended)
```

## SKILL.md format rules (CRITICAL)

- `name:` — kebab-case, matches the slash command: `/name`
- `description:` — **must be double-quoted** if it contains `[`, `]`, `{`, `}`, or `,` (YAML flow-indicator characters cause silent parse failure)
- `description:` — **keep under 200 characters** (AGY and Cursor have per-description limits; the working reference project uses ≤190 chars)
- Body — **keep under ~500 lines** (AGY truncates past this); for larger content use a `references/command_body.md` and `read_file` it from the SKILL.md

After editing any skill, re-run `scripts/setup-agy.ps1` and restart `agy`.

### Common mistakes (panel shows 0 commands / 0 skills)

| Mistake | Fix |
|---------|-----|
| `plugin.json` at `.agents/plugin.json` (root of `.agents`) | Remove it; use `.agents/plugins/<name>/plugin.json` only |
| Flat files `.agents/skills/scrape.md` next to `scrape/SKILL.md` | Keep **only** `{name}/SKILL.md` directories |
| `description:` contains `[...]` but is not double-quoted | Quote the whole value |
| `description:` over 200 chars | Trim it |
| Only ran `agy plugin validate` | Run `agy plugin install .agents/plugins/gemini_cli_testbed` or `scripts/setup-agy.ps1` |

## Slash commands (pipeline skills)

| Phase | Command |
|-------|---------|
| 1 | `/scrape url=... name=... [project=dmart-dloc]` |
| 2 | `/navigation-parser scraper=... [project=...]` |
| 3 | `/details-parser scraper=... [project=...]` |
| DHero | `/restaurant-details-parser`, `/menu-listings-parser`, `/menu-parser` |
| API | `/api-scrape`, `/api-navigation-parser`, `/api-details-parser` |
| Greenfield | `/greenfield-scrape url=... name=...` |
| Full run | `/run-pipeline project=... url=... name=... [kind=html\|api]` |

**Auto-chain spawns a fresh subprocess.** When `auto_next=true` (or via `/run-pipeline`), the skill closes the browser and spawns the next phase as `agent --yolo "/<next_phase> ..."` via `run_terminal_cmd`, then exits — state files are the only handoff between sessions.

## Cross-tool compatibility

`.agents/skills/<name>/SKILL.md` is the **universal agent skill format**:

| Tool | Global path synced by setup script | Activation |
|------|--------------------------------------|------------|
| **Antigravity CLI** | `~/.gemini/antigravity-cli/skills/` | `/<name>` slash command or semantic match |
| **Cursor** | `~/.cursor/skills/` | `/<name>` slash command or semantic match |
| **Claude Code** | n/a (uses `.claude/commands/`) | Referenced directly by operators |

Run `scripts/setup-agy.ps1` once per machine after any skill change; it pushes to all three paths.

## MCP: playwright-mod

### For Antigravity CLI (`agy`)

Configured in `.agents/mcp_config.json`. After setup, open **`/mcp`** in `agy` and confirm **playwright-mod** is listed.

### For Cursor CLI (`agent --yolo`) — HTTP mode required

Cursor CLI does **not** manage stdio MCP subprocesses standalone (that's the IDE's job). You must run playwright-mcp-mod as a persistent HTTP SSE server **before** launching `agent --yolo`.

**Step 1 — Start the server** (run once per machine session, in a separate terminal):

```powershell
pwsh -File scripts/start-playwright-mcp.ps1
```

This starts playwright-mcp-mod on `http://localhost:3000` (streamable HTTP, minimized window). The `.cursor/mcp.json` and `~/.cursor/mcp.json` are already configured to point to this URL.

**Step 2 — Launch Cursor CLI:**

```powershell
agent --yolo
```

The CLI will see playwright-mod as connected. Confirm with `/mcp`.

**To stop the server:** `Stop-Process -Name node -ErrorAction SilentlyContinue` (or close the minimized window).
