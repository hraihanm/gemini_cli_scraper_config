# Antigravity CLI (`agy`) setup

Google deprecated **Gemini CLI** on **2026-06-18**. This repo runs on **Antigravity CLI** (`agy`).

## Prerequisites

1. Install [Antigravity CLI](https://antigravity.google/docs/gcli-migration) (`agy` on PATH).
2. Log in inside `agy` (Google account).
3. Copy env template and set your API key:

```powershell
Copy-Item .agents\.env.example .agents\.env
# Edit AGY_API_KEY and AGY_MODEL (e.g. gemini-3.5-flash)
```

## One-time machine setup

From the repo root:

```powershell
pwsh -NoProfile -File scripts/setup-agy.ps1
```

Then **restart `agy`** in this directory:

```powershell
cd D:\DataHen\projects\gemini_cli_testbed
agy
```

### What the setup script does

| Step | Why |
|------|-----|
| Copies `.agents/skills/<name>/SKILL.md` → `~/.gemini/antigravity-cli/skills/` and `~/.gemini/skills/` | Knowledge skills available globally (semantic-loaded) |
| Copies `.agents/workflows/<name>.md` → `~/.gemini/antigravity-cli/workflows/` | Slash commands available globally |
| Stages `.agents/plugins/gemini_cli_testbed/{skills,workflows}/` + `agy plugin install` | Registers **playwright-mod** MCP and plugin commands |

> Running `agy` from the repo root already auto-discovers workspace `.agents/workflows/` and `.agents/skills/`. The global sync + plugin install make them available from any directory and register the MCP.

## The three layers (native model)

| Layer | Path | Role | Loaded |
|---|---|---|---|
| **AGENTS.md** | repo root | persona + firmware rules | always (prepended) |
| **Skills** | `.agents/skills/<name>/SKILL.md` | reusable knowledge | on-demand, semantic match on `description` |
| **Workflows** | `.agents/workflows/<name>.md` | slash commands that orchestrate & chain phases in one session | on `/<name>` |

## Layout (do not break)

```
.agents/
  mcp_config.json                  # workspace MCP (playwright-mod)
  .env                             # AGY_API_KEY, AGY_MODEL
  workflows/                       # slash commands (orchestration)
    scrape.md  navigation-parser.md  ...  run-pipeline.md
  skills/                          # knowledge only (one SKILL.md per topic)
    datahen-conventions/SKILL.md  selector-discovery/SKILL.md  ...
  plugins/
    gemini_cli_testbed/            # plugin bundle
      plugin.json                  # manifest (the ONLY plugin.json)
      mcp_config.json
      skills/   workflows/         # generated by setup-agy.ps1 (gitignored)

AGENTS.md                          # project context (always prepended)
```

### Common mistakes (panel shows **0 commands / 0 skills**)

| Mistake | Fix |
|---------|-----|
| `plugin.json` at **`.agents/plugin.json`** (root of `.agents`) | Remove it; use `.agents/plugins/<name>/plugin.json` only |
| Flat files `.agents/skills/scrape.md` next to `scrape/SKILL.md` | Keep **only** `{name}/SKILL.md` directories |
| Putting phase orchestration in skills | Orchestration belongs in `.agents/workflows/`; skills hold knowledge |
| Only ran `agy plugin validate` | Run **`agy plugin install .agents/plugins/gemini_cli_testbed`** or `scripts/setup-agy.ps1` |
| Never synced globals | Run `scripts/setup-agy.ps1` |

## Slash commands (pipeline workflows)

| Phase | Command |
|-------|---------|
| 1 | `/scrape url=... name=... [project=dmart-dloc]` |
| 2 | `/navigation-parser scraper=... [project=...]` |
| 3 | `/details-parser scraper=... [project=...]` |
| DHero | `/restaurant-details-parser`, `/menu-listings-parser`, `/menu-parser` |
| API | `/api-scrape`, `/api-navigation-parser`, `/api-details-parser` |
| Greenfield | `/greenfield-scrape url=... name=...` (forces `project=greenfield`) |
| Full run | `/run-pipeline project=... url=... name=... [kind=html\|api]` |

**Auto-chain is in-session.** When `auto_next=true` (or via `/run-pipeline`), the workflow runs the next phase in the **same `agy` session** via state files — no `chain.ps1`/`chain.sh` process spawn. On failure it stops and prints a resume line:

```text
agy --dangerously-skip-permissions --prompt-interactive "/<phase> scraper=... project=... auto_next=true"
```

## MCP: playwright-mod

Configured in `.agents/mcp_config.json`. After setup, open **`/mcp`** in `agy` and confirm **playwright-mod** is listed.

Build the sibling repo if needed:

```powershell
cd D:\DataHen\projects\playwright-mcp-mod
npm run build
```

## Verify

```powershell
agy plugin validate .agents/plugins/gemini_cli_testbed
# Expect: skills + workflows processed, mcpServers : 1 processed
```

In `agy`: type `/` and confirm the pipeline commands appear; the Skills panel should list the knowledge skills (datahen-conventions, selector-discovery, …).
